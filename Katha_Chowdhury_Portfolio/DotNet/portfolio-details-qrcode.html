Implementing QR Code Scanner with Google Authentication & Dynamic UI

Title: Implementing QR Code Scanner with Dynamic UI

Duration: 2 Days

Tools & Technologies Used
  • ASP.NET Core MVC (Visual Studio 2022)
  • JavaScript (for frontend interactivity)
  • Google OAuth (Authentication)
  • QRCode Generator Library (Back-end logic)
Goals & Objectives
  • The objective was to create a simple, secure, and interactive system for users
  • Enter user-defined information
  • Automatically generate a QR Code for that information
  • Authenticate using Google Login
  • Use a dynamic form that updates with a user-defined number of input fields (via JavaScript)
  • Ensure cross-device compatibility, especially with iPhone
  • This system needed to be lightweight, secure, and user-friendly — accessible both on desktop and mobile.
Problem Statement & Challenges
  • iPhone Rendering Issue -QR Code or result display was not working properly on iPhones.

Solution: Instead of directly showing results, a unique link was generated and shared. When the user clicked the link, the data loaded correctly and was displayed in a mobile-compatible format.

Google Authentication Integration
  • Needed secure and reliable login with minimal UI complexity.

Solution: Integrated Google OAuth for login via .NET Core MVC and ensured tokens were securely handled.

User Flow / Journey Map
  • User visits the QR Generator page.
  • User logs in using Google account.
  • User enters a number to define how many text inputs are needed.
  • The system dynamically renders the input fields (JavaScript logic).
  • After entering values, user clicks “Generate QR Code”.
  • A QR Code is generated and shown on screen.
  • On iPhone, a shareable link is provided to ensure the data is viewed in compatible format.
Results
  • Fully working QR Code generator with dynamic inputs.
  • Google login integration allowed secured access.
  • Improved client’s workflow by offering a reusable and portable QR system.
  • 100% mobile compatibility after solving iPhone rendering issue.
Reflection & Learnings
  • Cross-device UI testing is crucial even for small utilities.
  • Lightweight JavaScript interactions can massively improve user experience.
  • Quick projects still require scalability and flexibility in design.
  • Learned to handle mobile rendering issues and deep linking as fallback.
