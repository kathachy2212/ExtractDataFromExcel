Style-Wise Product Review Scraper for a Garments Company

Title: Style-Wise Product Review Scraper

Duration: 1 week

Tools & Technologies Used
  • Backend: ASP.NET Core Web API
  • Frontend: Angular (for dashboard display)
  • Scraping: Selenium with ChromeDriver
  • Automation: Hangfire for background job scheduling
Goals & Objectives
  • The client needed an automated, style-wise feedback tracking system that retrieves ratings, reviews, and sentiment insights from customer feedback to help improve their product performance and sales.
Client Requirements
  • Check customer reviews for each style of garment.
  • Collect ratings, positive and negative comments.
  • Detect trends over time.
  • Run automatically every day without manual effort.
Target Website Blocking
  • When requesting too many styles at once, the site blocked the scraping.

Solution:Limit scraping to 8 styles every 8 minutes, solving the issue by throttling requests.

Incomplete Comment Extraction
  • Comments were hidden behind “View More” buttons.

Solution:Programmatically click and scroll to load all reviews using Selenium before extraction.

Ratings Hidden Behind Interaction
  • Star ratings were available only after clicking the "Reviews" tab.

Solution:Used Selenium to interact with the page UI to reveal and scrape the ratings.

Daily Manual Execution Was Time-Consuming
  • Client didn't want to manually run the scraper every day.

Solution:Used Hangfire to schedule the scraping job daily at a specific time, automating the entire process.

User Flow / Journey Map
  • Admin uploads list of style IDs.
  • Hangfire automatically triggers scraper daily.
  • Scrapping target website using Selenium
  • Reports and data export
Results
  • 100% automated review collection daily.
  • Reduced manual monitoring time from hours to 0.
  • Gave client data-driven insights to adjust styles and improve products
  • Hangfire ensures consistent scheduling with job retries on failure.
Reflection & Learnings
  • Automating background tasks with a user-friendly dashboard made it easier for non-technical staff to use the system.
  • Learned how to handle real-world scraping issues like anti-bot measures and dynamic content loading.
Future upgrades could include
  • Email alerts when rating drops
